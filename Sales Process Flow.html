1
🔍 PROSPECT — หาข้อมูลร้านเป้าหมาย
ร้านอาหาร · บาร์ · โรงแรม · Craft Beer Bar · ร้านที่มีลูกค้ากลุ่มดื่มเบียร์
⏱ ทำทุกวัน (10+ ร้าน/สัปดาห์)
📌 สิ่งที่ต้องทำ
  • Search Google Maps: "bar", "restaurant", "hotel bar" + ย่าน
  • Search Wongnai: ร้านอาหาร/บาร์ที่มีเมนูเครื่องดื่ม
  • Facebook Group: Thai Craft Beer / ร้านอาหารใหม่
  • Cold Walk สำรวจย่าน 1 ย่าน/สัปดาห์
  • Foodpanda/GrabFood: ร้านที่มี Alcohol delivery
✅ ข้อมูลที่ต้องรวบรวมก่อน
  • ชื่อร้าน + ที่อยู่ + Zone
  • ประเภทร้าน (Bar/Restaurant/Hotel)
  • มี Tap System ไหม? (ดูจากรูป)
  • ชื่อผู้ติดต่อ (เจ้าของ/ผู้จัดการ)
  • เบอร์โทร / LINE / Facebook
✏️ ต้อง LOG ลง System
  • เพิ่ม Row ใหม่ใน Lead Pipeline
  • ระบุ Source (มาจากไหน)
  • ระบุวันที่เพิ่ม
  • Status = "Lead"
  • กำหนด Follow-up date
➡️
ผ่านไปขั้นถัดไปเมื่อ: มีข้อมูลร้านครบ (ชื่อ / ที่อยู่ / เบอร์ติดต่อ) และได้ประเมินเบื้องต้นแล้ว
ประเมินแล้วว่าร้านมีศักยภาพ
2
📋 QUALIFY — คัดกรองก่อนเข้าหา
ประหยัดเวลา · ไม่เข้าหาร้านที่ไม่มีศักยภาพ
⏱ ก่อน Cold Visit ทุกครั้ง
🔎 เช็คลิสต์ก่อน Approach
  • ร้านเปิดอยู่จริง? (ดู Google Maps reviews ล่าสุด)
  • มี Tap system หรือ Fridge craft beer?
  • ขนาดร้าน: มีที่นั่งพอสมควร?
  • Segment ตรงกับ craft beer audience?
  • อยู่ใน Delivery Zone ที่เราทำได้?
🎯 Target ร้านที่ดีที่สุด
  • มี Tap ≥ 2 ก๊อก อยู่แล้ว = Priority สูงสุด
  • ร้านอาหารที่มี Alcohol License + บาร์
  • Craft Beer Bar / Gastropub
  • Hotel Bar / Rooftop Bar / Pool Bar
  • Event Space / Co-Working ที่มี Bar
  • ร้านที่ลูกค้าอายุ 25-45 ปี และดื่มเบียร์
✅
Qualified: มี Tap + Volume + อยู่ใน Zone → ไป Approach
❌
Not Qualified: ไม่มี Tap / ไม่ใช่กลุ่มเป้าหมาย → Status = "Not Interested" + บันทึกเหตุผล
✏️ ต้อง LOG ลง System
  • กรอก Qualify fields ใน Lead Pipeline (Tap: Yes/No, Est.Volume, ประเภทร้าน)
Qualified แล้ว — พร้อมเข้าหา
3
🤝 APPROACH — เข้าหาครั้งแรก
เป้าหมายของขั้นนี้คือนัด Tasting เท่านั้น — ยังไม่ต้องขาย
⏱ ภายใน 3 วันหลัง Qualify
📞 วิธีเข้าหา
  • Cold Visit: แวะร้าน 14:00-17:00 น. (ก่อน peak)
  • Cold Call: โทรนัด ใช้ Script ด้านล่าง
  • LINE: Add LINE ร้าน ส่งข้อความสั้น
💬 Script เปิดการสนทนา
  • "สวัสดีครับ ผมมาจาก 12 Plato Brewing เบียร์คราฟท์กรุงเทพ"
  • "อยากแวะมาแนะนำ product ให้ทราบครับ มีเวลาสักครู่ไหม?"
  • "ขอนัดให้ทีมเราเอาตัวอย่างมาให้ชิมได้ไหมครับ?"
✏️ ต้อง LOG ลง System
  • Activity Log: ประเภท = "Cold Visit" หรือ "Cold Call"
  • บันทึกผลการติดต่อ
  • Update Status = "Contacted"
  • กำหนด Follow-up date ถัดไป
➡️
สนใจ + ยินดีรับนัด → Update Status "Visited" → ไป Tasting
🔄
ยังไม่สะดวก → กำหนด Follow-up ใหม่ ห้ามเกิน 7 วัน
นัด Tasting สำเร็จ
4
🍺 PITCH & TASTING — นำเสนอ + ให้ชิม
ให้ลูกค้า "รู้สึก" กับเบียร์ก่อน — ข้อมูลตามหลัง
⏱ ภายใน 5 วันหลัง Approach
🎒 เตรียมไปด้วย
  • ตัวอย่างเบียร์ 2-3 style (แบบ 330ml)
  • Pricelist (PDF/HTML Mobile)
  • ข้อมูลสินค้า: ABV, IBU, Style, Description
  • นามบัตร / LINE QR Code
🗣️ จุดขายที่ต้อง Present
  • 14 SKU Year-Round + 34 Seasonal
  • MOQ = 1 KEG เท่านั้น (ไม่มีขั้นต่ำ)
  • Delivery 1-2 วันทำการ
  • Price: 5,300 - 6,300 ฿/KEG (20L)
  • ผลิตในกรุงเทพฯ สด ไม่ผ่าน Distributor
✏️ ต้อง LOG ลง System
  • Activity Log: ประเภท = "Tasting"
  • บันทึก Style ที่ชิม + ความเห็นลูกค้า
  • Update Status = "Tasting"
  • Follow-up ภายใน 48 ชั่วโมง
➡️
ลูกค้าสนใจและเริ่มถามเรื่องราคา/เงื่อนไข → Update Status "Negotiating" → ไป Negotiate
ลูกค้าสนใจสั่งซื้อ
5
💬 NEGOTIATE — ต่อรอง
ยืนราคา Pricelist · ห้ามลดโดยไม่ได้รับอนุมัติจากเจ้าของ
⏱ ปิดภายใน 7 วัน
❓ คำถามที่มักเจอ
  • ลดราคาได้ไหม? → "ราคานี้รวม VAT แล้ว ไม่มีขั้นต่ำครับ"
  • จ่ายยังไง? → ตอบตามนโยบายบริษัท
  • Shelf life? → แจ้งตามสินค้าแต่ละ SKU
  • ลอง 1 KEG ก่อนได้ไหม? → "ได้เลยครับ MOQ 1 KEG"
🚫 ห้ามทำโดยเด็ดขาด
  • ลดราคาโดยไม่ได้รับอนุมัติ
  • สัญญาเงื่อนไขพิเศษโดยไม่แจ้งเจ้าของ
  • รับ Order โดยไม่ confirm stock
  • อ้างข้อมูลสินค้าที่ไม่แน่ใจ
✏️ ต้อง LOG ลง System
  • Activity Log: ประเภท = "Negotiation"
  • บันทึกเงื่อนไขที่ลูกค้าขอ
  • Update Status = "Negotiating"
  • แจ้งเจ้าของถ้ามีเงื่อนไขพิเศษ
✅
ตกลงได้ → ไป Close + รับ Order
❌
ไม่ตกลง → Status "Not Interested" + บันทึกเหตุผล ค่อยติดตามอีก 3 เดือน
ตกลงราคาและเงื่อนไขแล้ว
6
✅ CLOSE + FIRST ORDER — ปิดการขาย
ขั้นตอนนี้สำคัญที่สุด — ต้องเป็นระบบ ไม่ลืมขั้นตอน
⏱ ยืนยัน Order ภายใน 24 ชั่วโมง
📋 ขั้นตอนปิดการขาย
  • 1. Confirm stock กับทีม 12 Plato ก่อน
  • 2. ส่ง Order Form / ใบเสนอราคา
  • 3. ยืนยัน Delivery Date กับลูกค้า
  • 4. แจ้งทีม Operation รับ Order
  • 5. รับการชำระเงิน (ตามนโยบาย)
📦 สิ่งที่ต้องแจ้งลูกค้า
  • KEG Size: 20L เท่านั้น
  • Delivery: 1-2 วันทำการ
  • ต้องมี Tap System รองรับ
  • แนะนำ Cleaning schedule
  • แจ้ง Contact สำหรับ Reorder
✏️ ต้อง LOG ลง System
  • Update Lead Pipeline Status = "Customer"
  • เพิ่มข้อมูลใน Customer Master Sheet
  • บันทึก Volume + KEG brands ที่สั่ง
  • Activity Log: ประเภท = "Order Taken"
🎉
Order ยืนยันแล้ว + บันทึกใน Customer Master ครบ → ไป Retain (Follow-up หลังส่งของ)
ส่งของแล้ว
7
💎 RETAIN — รักษาลูกค้า
ค่าใช้จ่ายในการหาลูกค้าใหม่ > ค่าใช้จ่ายในการรักษาลูกค้าเก่า 5 เท่า
⏱ Follow-up สม่ำเสมอทุกเดือน
📅 Timeline Follow-up
  • +3 วัน: "เบียร์เป็นอย่างไรบ้างครับ?"
  • +2 สัปดาห์: แนะนำ SKU ใหม่ / Seasonal
  • ทุกเดือน: Update menu + ถาม Reorder
  • ทุก 3 เดือน: เยี่ยมร้าน + ดู Tap condition
📈 สัญญาณที่ต้องเฝ้าระวัง
  • ไม่ Reorder เกิน 6 สัปดาห์ = At Risk
  • ลด Volume ลงกว่า 30% = ต้องโทรถาม
  • เปลี่ยน Contact person = ต้อง Re-intro
  • มี Complaint → แจ้งทีมทันที
✏️ ต้อง LOG ลง System
  • Activity Log ทุกครั้งที่ติดต่อ
  • Update Customer Master: Volume + Last Order Date
  • Update Status: Active / At Risk / VIP

⚡ กฎ Accountability — ไม่มีข้อยกเว้น

Rule 01
ไม่มีข้อมูลใน System = ไม่ถือว่าทำงาน
ทุก Visit, Call, Tasting ต้องมี Log ใน Activity Sheet ภายในวันนั้น
Rule 02
Lead ทุกตัวต้องมี Next Follow-up Date
ถ้า Lead ค้างอยู่ใน Pipeline เกิน 7 วันโดยไม่มี Action = ต้องอธิบาย
Rule 03
Weekly Report ส่งทุกวันศุกร์ 17:00 น.
ระบุ: Activity ที่ทำ / Top 3 Prospects / ติดขัดอะไร / แผนสัปดาห์หน้า
Rule 04
ห้ามลดราคาโดยไม่ได้รับอนุมัติ
ทุกเงื่อนไขพิเศษต้องแจ้งเจ้าของก่อนสัญญากับลูกค้า
Rule 05
ข้อมูลลูกค้าทั้งหมดเป็นของบริษัท
Contact ทุกตัวต้องอยู่ใน System บริษัท ไม่ใช่ในมือถือส่วนตัว
Rule 06
New Leads ต้องมาจากการหาใหม่เท่านั้น
Contact จากงานเก่าไม่นับ จนกว่าจะถูก Record ลง System พร้อมระบุ Source
